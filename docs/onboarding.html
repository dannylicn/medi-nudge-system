Patient Onboarding Journey

Medi‑Nudge System — WhatsApp-first, with optional web capture for OCR and voice enrolment

Primary Journey (WhatsApp-first)

Entry1) Enrollment triggerCare team enrolls patient (clinic) or patient self-initiates via QR/link.
Consent2) WhatsApp opt-in + consentPatient agrees to receive medication reminders and to share messages for care follow-up.
Identity3) Verify phone & languageConfirm preferred language (en/zh/ms/ta) and confirm it's the patient (basic verification).
Meds4) Add medication listPick one: (a) upload sticker/photo for OCR, (b) manual entry, (c) import from dispensing feed.
Review5) Confirm extracted medsPatient (and/or care coordinator) confirms name/dose/frequency and refill interval.
Schedule6) Set reminder preferencesAllowed windows, frequency (refill reminders vs dose reminders), quiet hours.
Optional7) Voice reminders (optional)Invite loved-one voice donor → dual consent → sample upload → test playback approval.
Done8) Welcome + first check-inSend a welcome message and a short “reply guide” (YES/NO/HELP/SIDE EFFECTS).

Design intent: minimize friction; if OCR/voice fails, fall back to text + manual review.

Artifacts Created

Minimum to start nudging

  • Patient: phone, language, consent/active flag
  • PatientMedication: medication, dosage, refill interval
  • DispensingRecord: last dispense date + days_supply (from feed or confirmed OCR)

Optional enhancements

  • PrescriptionScan + ExtractedMedicationField (OCR)
  • VoiceProfile + VoiceSample (voice clone)

WhatsApp message guide

  • Confirm: “YES”, “Collected”, “Done”
  • Question: “HELP …”
  • Side effects: “SIDE EFFECT …” (auto-escalate)

Two Entry Modes

A) Clinic-enrolled (recommended for v1)

  • Care coordinator creates patient with phone + language (and optionally imports dispensing feed).
  • System sends a WhatsApp invite: consent + simple verification + onboarding steps.
  • OCR/voice steps are optional; coordinator can review OCR fields before confirming.

B) Self-enrolled (QR/link)

  • Patient scans QR at clinic/pharmacy → opens web page to enter phone, then continues on WhatsApp.
  • Web page can collect the first prescription photo to reduce WhatsApp friction.
  • Care team review required before medication records are committed.
